# PII Scanner 시장 분석 보고서

## 1. 개요
본 문서는 "PC 개인정보 탐지 솔루션"의 성공적인 시장 진입을 위해 국내외 주요 경쟁사 분석, 기술 트렌드 파악, 그리고 차별화 전략을 수립하는 것을 목적으로 한다.

**핵심 타겟:**
- 기존 DLP 솔루션의 무거운 에이전트와 성능 저하로 고통받는 기업
- 높은 오탐율로 인해 관리 비용이 과다하게 발생하는 조직
- 한국어 비정형 데이터(이름, 주소 등)의 정밀한 탐지가 필요한 공공/금융 기관

## 2. 한국 시장 분석 (주요 플레이어)

국내 시장은 컴플라이언스(개인정보보호법) 대응을 중심으로 성숙기에 접어들었으나, "정확도"와 "성능" 측면에서는 여전히 고객 불만이 존재함.

| 제조사 | 주요 솔루션 | 강점 (Strengths) | 약점/불만 요소 (Weaknesses) |
| :--- | :--- | :--- | :--- |
| **소만사** (Somansa) | Privacy-i, WebKeeper | - 압도적인 시장 점유율 (1위)<br>- 엔드포인트부터 네트워크까지 통합 라인업<br>- 오랜 업력으로 축적된 패턴 DB | - 에이전트가 무겁고 시스템 리소스 점유율이 높다는 인식<br>- 레거시 코드로 인한 유지보수 복잡성 가능성 |
| **컴트루테크놀로지** | 셜록홈즈 PC정보보안 | - 'AI' 마케팅 선점 (이미지 OCR, AI 스캔)<br>- 공공기관 레퍼런스 다수 확보<br>- 다양한 형태(USB, 구축형) 제공 | - AI 기능 활성화 시 클라이언트 부하 증가 우려<br>- UI/UX가 다소 노후화된 경향 |
| **파수** (Fasoo) | Fasoo Data Radar | - DRM 기술 기반의 강력한 문서 암호화 연동<br>- 데이터 식별 및 분류 체계가 정교함 | - DRM과 연동되므로 도입 비용이 높고 구조가 복잡<br>- 단순 PII 탐지만 원하는 고객에게는 오버스펙 |
| **지란지교데이터** | PCFILTER | - 자산관리 등 부가 기능 통합<br>- 상대적으로 가벼운 도입 장벽 | - 정밀한 비정형 데이터(문맥 인식) 탐지 능력은 상위 경쟁사 대비 검증 필요 |

글로벌 솔루션은 "통합 플랫폼"화 되고 있으나, **한국어 특화 처리**와 **엔드포인트 경량화** 측면에서 틈새가 존재함.

| 제조사 | 솔루션 | 특징 및 강점 | 한국 시장에서의 한계 |
| :--- | :--- | :--- | :--- |
| **Symantec** (Broadcom) | Symantec DLP | - 글로벌 표준, 가장 강력한 탐지 엔진<br>- 방대한 정책 라이브러리 | - **무거움:** "PC가 느려진다"는 불만의 대명사<br>- 한국어 비정형 패턴(주소, 이름) 탐지 정확도 낮음<br>- 도입 및 유지보수 비용 매우 높음 |

## 4. 시장 진입 전략 및 차별화 포인트

기존 시장은 **"탐지했는가?"**에 집중했지만, 이제는 **"얼마나 정확하게, 방해 없이 탐지하는가?"**가 핵심 경쟁력이다.

### 4.1 차별화 포인트 (USP)

1.  **"Zero-Interruption" Performance (성능)**
    *   **경쟁사:** Java/C++ 기반의 무거운 에이전트, 전체 스캔 시 CPU 100% 점유.
    *   **우리 솔루션:** **Rust** 언어의 메모리 안전성과 고성능 활용. **ONNX Runtime**을 이용한 경량 AI 추론으로 GPU 없이도 백그라운드에서 조용히 동작.

2.  **Context-Aware Precision (정확도)**
    *   **경쟁사:** 단순 정규식(Regex) 매칭으로 "010-1234-5678"만 찾음. (오탐 높음)
    *   **우리 솔루션:** **KoELECTRA 기반 NER(개체명 인식)** 모델 적용. 문맥을 파악하여 단순 숫자 나열과 실제 전화번호를 구분. "서울시 강남구"와 같은 주소 및 한국인 이름 탐지 탁월.

3.  **Real-time & Incremental (실시간성)**
    *   **경쟁사:** 예약된 스케줄에 전체 스캔 (Disk I/O 과부하).
    *   **우리 솔루션:** 파일 시스템 이벤트(FSE) 후킹으로 **변경된 파일만 즉시 스캔**. 증분 검색 지원으로 평상시 리소스 사용량 '0'에 수렴.

### 4.2 진입 전략 (Go-to-Market)

*   **1단계 (틈새 공략):** 기존 무거운 DLP에 지친 **개발자/IT 기업** 타겟. "개발자 PC에서도 느려지지 않는 보안 솔루션"으로 포지셔닝.
*   **2단계 (Win-Back):** 오탐/과탐으로 인해 보안팀 리소스 낭비가 심한 기업에게 **"오탐률 99% 감소"**를 수치로 증명하며 교체 수요 공략.
*   **3단계 (SaaS):** 중소기업(SMB)을 위한 구독형 클라우드 관리 모델 제공. 초기 구축 비용 제거.

## 5. 아키텍처 제언

경쟁사 분석을 토대로 본 프로젝트에 적합한 아키텍처 방향성을 제안함.

*   **Client (Agent):**
    *   언어: **Rust** (필수 - 메모리 안전성 및 퍼포먼스)
    *   AI 엔진: **ONNX Runtime** (PyTorch 의존성 제거, 바이너리 용량 최소화)
    *   탐지 파이프라인: `Filter(확장자/헤더) -> Regex(1차 선별) -> NER(2차 정밀 검증)` 구조로 CPU 부하 최소화.
*   **Server:**
    *   Go 또는 Rust 기반의 고성능 API 서버.
    *   대규모 로그 처리를 위한 Kafka/Elasticsearch 스택 고려 (향후 5만 대 규모 확장 대비).
*   **Update Strategy:**
    *   바이너리 전체 업데이트가 아닌, **탐지 룰(Regex) 및 AI 모델(ONNX 파일)**만 별도로 핫스왑 가능한 구조 설계.

## 6. 결론
현재 시장은 "기능"은 포화 상태이나 "품질(성능/정확도)"은 여전히 미충족 상태임.
**Rust + AI(NER)** 기술 스택은 기존 레거시 보안 업체들이 쉽게 따라올 수 없는 강력한 기술적 해자(Moat)가 될 것임.
